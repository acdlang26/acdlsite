Language Reference

Complete reference for the Agentic Context Description Language. ACDL declaratively specifies prompt structures sent to LLMs, separating structural concerns from content and implementation.

Specification Structure

An ACDL specification defines a named prompt template parameterized by optional indices. The body consists of an ordered sequence of prompt blocks—role messages, label blocks, and control flow constructs:

PromptName[idx1, idx2, ...]: {
    <prompt-blocks>
}

A single file may contain multiple specifications, each defining a separate prompt template:

PromptName1[idx1, idx2, ...]: {
    <prompt-blocks>
}

PromptName2[idx1, idx2, ...]: {
    <prompt-blocks>
}

Role Messages

Every message carries exactly one role. Four roles serve the chat format; a fifth pseudo-role serves the legacy completion format:

MarkerPurpose
S:System—instructions, persona, tool descriptions, behavioral constraints
U:User—external input, observations, tool results
A:Assistant—prior model outputs, reasoning traces, chosen actions
T:Tool—structured tool call results
N:None—single unstructured text block (completion format only)

The multi-line form encloses content in braces and permits any combination of content elements and control flow:

S: {
    INSTRUCTIONS
    AVAILABLE_TOOLS
    env.datetime[@t]
}

The single-line form omits braces and accepts exactly one content element:

U: env.user_question[@t]
A: resp.answer[@t]
S: INSTRUCTIONS

Completion Format: The N: role imposes two constraints: (1) exactly one N: block may appear per prompt, and (2) no chat roles may appear in the same specification.

Scoping Rules

The language enforces a strict two-level scope:

  • Top level: only role messages, label/marker blocks, control flow, name definitions, and comments
  • Inside role blocks: context variables, functions, templates, control flow, comments, name definitions, marking blocks

Role messages may not appear inside other role messages.

Context Variables

Context variables reference dynamic runtime data:

namespace.path[indices]
NamespaceUse Cases
envEnvironment—external inputs, observations, user queries
sysSystem—agent state, memory, tool configurations
respResponse—prior LLM outputs, reasoning traces
promptPrompt—content from labeled sections
env.user_question[@T]           // user question at time T
sys.agent_desc                   // constant agent description
sys.tool[@t].tool_response      // tool response at time t
prompt.History[@T-1]            // history label at previous step

Indices

Time Indices

The @ symbol denotes the primary time dimension. Current time uses capitals (@T); iteration uses lowercase (@t):

@T      // Current time step
@T-1    // Previous time step
@0      // Initial step
@T.I    // Current substep of current turn
@t.i    // Substep i of turn t (in loops)

Examples

// Iterating over all previous turns
ForEach(@t: range(1, @T-1)) {
    env.observation[@t]
}

// Nested: substeps within each previous turn
ForEach(@t: range(1, @T-1)) {
    ForEach(i: range(0, @t.substeps)) {
        sys.action[@t.i]
    }
}

Non-Time Indices

Non-time indices address other dimensions:

[sys.agent_name]
[bomb]

Templates

Templates are ALL_CAPS placeholders representing text blocks:

INSTRUCTIONS           // Task explanation
AVAILABLE_TOOLS        // Tool list
QUERY(sys.agent_name) // Parameterized template

Functions

Functions represent computed content—summarization, retrieval, formatting. Names use camelCase:

summarize(prompt.History[@t])
k_relevant_docs(env.user_input[@T])
range(1, @T-1, 2)

The built-in range(start, stop, step) generates sequences for ForEach loops (inclusive).

Control Flow

ForEach

ForEach(@t: range(1, @T-1)) {
    U: env.user_question[@t]
    A: resp.answer[@t]
}

If / ElseIf / Else

If (@T > 1) {
    ForEach(@t: range(1, @T-1)) {
        U: env.user_input[@t]
        A: resp.answer[@t]
    }
}

Switch / Case / Default

Switch (sys.action_type[@t]) {
    Case "search" {
        U: env.search_results[@t]
    }
    Case "calculate" {
        U: env.calculation[@t]
    }
    Default {
        U: env.fallback[@t]
    }
}

Early Termination

Signals the prompt ends at a given condition:

PromptEndsHere when (@T == @t && @T.0)

Label Blocks

Groups prompt blocks under a name for later reference:

Prompt[@t]: {
    Setup {
        S: INSTRUCTIONS
        U: env.initial_context[@0]
    }
    History {
        ForEach(@t: range(1, @T-1)) {
            U: env.user_question[@t]
            A: resp.answer[@t]
        }
    }
}

Mark Blocks

Annotates sections for visual emphasis in rendered output:

MARK 1 {
    <prompt-blocks>
}

Name Definitions

Binds a symbolic name to an expression:

Name docs := k_relevant_docs(env.user_input[@T])
ForEach(i: range(1, $docs.len)) {
    $docs[i].source
    $docs[i].content
}

List Comprehensions

Name summaries := [sys.summary[@t] for t in range(@T, @T-900, 100)]

Comments

S: {
    // This comment appears on its own line
    INSTRUCTIONS   // Renders beside INSTRUCTIONS
}
