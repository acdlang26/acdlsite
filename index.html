What is ACDL?

Agentic Context Description Language (ACDL) is a domain-specific language for describing how LLM contexts are constructed as the state of the system evolves over time. An ACDL specification describes the structure of the prompt presented to the model at each interaction step: which messages appear, in what order, with what content, and under what conditions.

The language is descriptive—it specifies what the context contains, not how it is computed—and is independent of any particular codebase or runtime framework. ACDL captures the structural blueprint of a prompt rather than its exact wording.

The Problem

  • No standard way to describe context structure
  • Informal prose leaves room for misinterpretation
  • Hard to compare systems or reproduce results
  • Difficult to communicate context evolution over time

ACDL Provides

  • Precise, unambiguous specifications
  • Visual renderings for easy comparison
  • Implementation-agnostic descriptions
  • Time-indexed evolution of context structure

Language Example

A basic RAG (Retrieval-Augmented Generation) prompt described in ACDL:

BasicRAG[@T]: {
    S: INSTRUCTIONS
    U: {
        Name docs := k_relevant_docs(env.user_input[@T])
        ForEach(i: range(1, $docs.len)) {
            $docs[i].source
            $docs[i].content
        }
        ANSWER_Q_FROM_DOCS
        env.user_input[@T]
    }
}

This specification shows a system message with instructions, followed by a user message containing retrieved documents and the user's question. The k_relevant_docs function retrieves documents based on the current user input.

Key Language Features

S: Role Messages

Four chat roles (System, User, Assistant, Tool) plus a completion format. Each message carries exactly one role with visual color-coding.

@T Time Indexing

Explicit time indices describe how contexts evolve. @T represents the current step, with support for sub-steps (@T.I) and history iteration.

env Context Variables

Four namespaces: env (environment), sys (system state), resp (LLM responses), and prompt (labeled sections).

If Control Flow

ForEach loops, If/ElseIf/Else conditions, and Switch/Case constructs determine what appears in the context based on runtime state.

ABC Templates

ALL_CAPS placeholders represent text blocks. They separate prompt architecture from prose, with optional arguments for parameterization.

fn() Functions

Named functions represent computed content—summarization, retrieval, or any transformation that can't be a simple variable lookup.
